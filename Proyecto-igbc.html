Portal IGBC - App de Gestión

Aplicación móvil multiplataforma para la gestión integral de ventas y administración

.NET MAUI 8.0 Android | iOS | Windows Estado: Activo

AndroidAppIGBC (Portal IGBC) es una aplicación móvil robusta desarrollada con .NET MAUI (.NET 8). Implementa una arquitectura MVVM y gestiona múltiples roles de usuario (Vendedor, Aprobador, Administrador) para cubrir todo el ciclo de venta, desde la toma del pedido hasta su aprobación y análisis.

Características Principales

Seguridad y Autenticación

  • Login seguro: Autenticación basada en JWT Tokens.
  • Roles Dinámicos: Navegación personalizada (Shells separados) según si es Admin, Vendedor o Aprobador.
  • Sesión: Cierre de sesión automático ante credenciales revocadas.

Módulo de Vendedor

  • Catálogo: Visualización optimizada con caché de imágenes (FFImageLoading).
  • Carrito: Validación de stock en tiempo real.
  • Offline: Base de datos local (SQLite) para gestión de clientes sin conexión.
  • Historial: Estado de cuenta y cobranza.

Módulo de Aprobador

  • Dashboard: Métricas de pedidos pendientes.
  • Revisión: Detalle profundo de cada pedido.
  • Acciones: Editar, Aprobar o Rechazar pedidos en tiempo real.

Módulo de Administrador

  • Dashboard Global: Estadísticas de ventas y cuentas por cobrar.
  • Fuerza de Ventas: Monitoreo de ubicación y desempeño de vendedores activos.

Tecnología y Servicios Avanzados

  • Notificaciones Push: Integración completa con Firebase Cloud Messaging.
  • Tiempo Real (SignalR): Actualizaciones inmediatas de estado de pedidos y alertas.
  • Segundo Plano: Servicios para monitoreo de stock mientras se crea el pedido.

Stack Tecnológico

C# .NET MAUI 8.0 MVVM SQLite SignalR Firebase FFImageLoading

Configuración

Prerrequisitos

  • Visual Studio 2022 (Workload .NET MAUI).
  • .NET 8.0 SDK.
  • Emulador Android o dispositivo físico.

Configuración de API

Es necesario apuntar la app al servidor backend:

// Archivo: Services/Link_Api.cs public static class Link_Api { // Cambia esto por la IP de tu servidor o DevTunnel public static string Link_Server = "https://tu-api-url.com"; }

Configuración Firebase

Para que las notificaciones funcionen en Android:

  • Colocar el archivo google-services.json en la ruta Platforms/Android/.
  • Verificar el package_name en AndroidManifest.xml.

Estructura del Proyecto

AndroidAppIGBC/ ├── Clases_Api/ -> Modelos DTO compartidos. ├── Definiciones/ -> Constantes y configs. ├── Models/ -> Modelos SQLite y UI. ├── Services/ -> Lógica (API, Auth, SignalR). ├── ViewModels/ -> Lógica de presentación (MVVM). │ ├── ViewModelsAdm/ │ └── ViewModelsAprobador/ ├── Views/ -> Interfaz XAML. ├── Platforms/ -> Código nativo (Android/iOS). └── Resources/ -> Estilos, imágenes, fuentes.

Notas de Desarrollo

  • AppShell Dinámico: La app inicia en una LoadingPage que decide qué Shell cargar según el rol.
  • Dev Mode: Existe una página oculta para desarrolladores para probar conexiones y bases de datos locales.
